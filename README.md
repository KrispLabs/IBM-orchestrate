# IBM-orchestrate
Whenever we write code, we’re supposed to also write tests to make sure it works properly. But in reality, most developers either skip tests or don’t update them when the code changes. That leads to bugs and things breaking later.

So our idea is to build a system that does all of this automatically. It reads your code, understands what it’s supposed to do, and creates the tests for you. Then whenever you change the code, it automatically updates those tests as well.

We’re using IBM Bob to act like a smart assistant that understands the code and keeps everything in sync.

Basically, we’re removing the headache of writing and maintaining tests, so developers can focus on building features while the system handles reliability in the background.”
